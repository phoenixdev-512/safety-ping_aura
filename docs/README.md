## Documentation Standards

### Writing Style

- **Professional**: No emojis, informal language, or slang
- **Clear**: Use simple, direct language
- **Concise**: Avoid unnecessary words
- **Consistent**: Use consistent terminology throughout
- **Structured**: Use headings, lists, and formatting effectively

### Formatting

- Use Markdown formatting consistently
- Include table of contents for documents over 500 lines
- Use code blocks for commands and code examples
- Include line breaks between sections
- Use descriptive headings

### Maintenance

- Update version numbers when releasing
- Keep CHANGELOG.md current
- Review and update documentation with major changes
- Ensure all links work and point to correct locations
- Update screenshots when UI changes significantly
